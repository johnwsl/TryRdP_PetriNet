2-Modelagem Básica

  1. 2.2 Redes Elementares (Distribuição)

      Imagine um cenário onde um modem recebe um sinal e esse mesmo sinal é transmitido simultaneamente para dois roteadores. Esses roteadores, por sua vez, transmitem esses sinais para computadores. O modelo que representa esse cenário é mostrado abaixo. No Estado 1, o modem ainda não recebeu o sinal. No Estado 2, o modem recebe o sinal (representado por um token no lugar Modem) após o disparo da transição ChegadaDoSinal. No Estado 3, ocorre a execução da transição SaídaParaRoteador. Isto é, o modem envia o sinal simultaneamente para o Roteador1 e o Roteador2. Vale lembrar que a pré-condição para que a transição SaídaParaRoteador seja executada é haver um token no lugar Modem (Estado 2). Quando a transição SaídaParaRoteador é disparada, como resultado (pós-condição), ela consome um token do lugar Modem e deposita um token nos lugares Roteador1 e Roteador2 (Estado 3). Por fim, as transições Saída1 e Saída2 podem ser executadas, indicando que os roteadores enviaram os sinais para computadores. Note que, após isso, o modelo retorna para o Estado 1.

      Distribuição: o exemplo mencionado acima representa o funcionamento de uma rede com o padrão de distribuição. Esse padrão possibilita a criação de processos paralelos a partir de um processo prévio. Como mostrado no exemplo, após o sinal chegar no modem, ele é enviado ao mesmo tempo (em paralelo) para dois roteadores.
